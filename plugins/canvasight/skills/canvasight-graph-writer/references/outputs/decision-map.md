# Output: decision-map

Use one decision node branching to criteria/constraints and viable options, then converge on the result.

- Give options parallel visual weight before evaluation.
- Keep independently selectable options and independently testable criteria as nodes; retain tightly coupled rationale together with the decision it explains.
- Connect evidence and trade-offs to the option they affect.
- Keep selected and rejected outcomes explicit.
- Do not connect a decision as final when a required input is still missing; use a blocker/question node instead.

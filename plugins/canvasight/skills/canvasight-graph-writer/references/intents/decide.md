# Intent: decide

Use when the user needs a choice, prioritization, trade-off, or recorded decision.

## Thinking contract

- Define the decision and the constraints that make it necessary.
- Include realistic options, including the status quo when relevant.
- Use explicit evaluation criteria and evidence; distinguish facts from preferences.
- Record trade-offs, risks, reversibility, and why rejected options lost.
- End with a decision or a precisely stated missing input that prevents one.

Normally pair with `maturity: decide` and `output: decision-map`.

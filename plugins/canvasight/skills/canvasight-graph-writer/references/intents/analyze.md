# Intent: analyze

Use for explaining an existing repository, product, document, behavior, strategy, or result.

## Thinking contract

- Inspect the object or supplied evidence before drawing conclusions.
- Label three epistemic states in node bodies: **Confirmed**, **Inference**, and **Question**.
- Show structure and relationships, then explain causes, strengths, weaknesses, risks, and implications.
- Link conclusions to evidence nodes or name the inspected source in the same node.
- Do not turn analysis into a speculative redesign unless the user asks for recommendations.

If evidence cannot be accessed, create a bounded question or blocker rather than inventing details.

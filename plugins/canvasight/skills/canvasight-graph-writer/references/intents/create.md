# Intent: create

Use for building a new concept, product, design, outline, or solution from an initial request.

## Thinking contract

- State the intended outcome and the change it should create before proposing structure.
- Separate confirmed user input from assumptions introduced to make progress.
- Establish users or audience, scope, constraints, and success before implementation detail.
- Generate questions where missing information changes the direction; do not disguise them as decisions.
- Keep technical implementation subordinate unless the primary domain or user explicitly requires it.

Combine this intent with the complete primary-domain contract. `create` does not itself imply a new Page: refining a newly discussed idea on an active Page still uses `merge-active-page`.

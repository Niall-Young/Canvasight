# Intent: execute

Use for turning a defined direction into sequenced work, deliverables, and verification.

## Thinking contract

- Start from confirmed scope and completion criteria, not a generic task list.
- Identify prerequisites, dependencies, parallelizable branches, ownership if known, and stop conditions.
- Attach a concrete verification method to every major stage.
- Include rollback or recovery for risky or externally visible changes.
- Mark work as complete only from current evidence; otherwise use planned, in-progress, blocked, or unknown language.

Normally pair with `task-execution`, `maturity: deliver`, and `output: execution-plan`.

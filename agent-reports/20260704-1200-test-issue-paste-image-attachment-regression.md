# 图片粘贴无法直接进入节点
## 发现者
测试监督 Agent
## 现象
Canvasight 网页画布中，用户将图片复制到剪贴板后无法通过粘贴直接添加到当前节点附件。
## 影响范围
影响节点附件输入工作流。上传按钮和拖拽附件入口仍存在，但从截图、剪贴板图片快速补充上下文的核心能力失效。
## 复现步骤
1. 打开 Canvasight 网页画布并选择一个任务节点。
2. 复制一张图片到系统剪贴板。
3. 在画布或节点中执行粘贴。
4. 观察节点附件列表没有新增图片。
## 证据
代码搜索显示 `addFilesToNode(..., "paste")`、`AttachmentSource = "paste"` 和附件保存 API 仍存在，但 `plugins/canvasight/src` 中没有 `onPaste` 或 clipboard paste 监听入口。
## 初步归因
客户端迁移为网页插件时保留了附件保存链路，但漏迁移了图片粘贴事件入口。

开发 Agent
